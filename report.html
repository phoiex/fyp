2. Abstract

This report outlines the development progress and goals of the "Web Development for Shopping Website" project. The main objective of this project is to create an efficient, user-friendly e-commerce website, addressing critical issues in existing e-commerce platforms, such as slow page load times, complex navigation, and insufficient payment integration. Currently, the backend API development and database integration are complete. The next phase will focus on frontend UI/UX design, security compliance, and performance optimization. The report details the project background, requirements analysis, technical architecture, development progress, and future plans.

3. Introduction

Background: As online shopping continues to grow, e-commerce platforms face challenges such as high traffic, complex product management, and varied payment requirements. The background of this project is to address the common problems found in existing shopping websites, such as slow page loading, poor user experience, and insecure payment processes. The aim is to build a robust platform that can handle these challenges efficiently.

Project Goals & Significance: The core goal of the project is to develop a fully-functional, user-friendly, and responsive shopping website. The website will include user account management, product listing, shopping cart functionality, and secure payment integration. This project will improve the online shopping experience by enhancing site performance, ensuring secure transactions, and streamlining the checkout process, which will lead to higher customer satisfaction and improved sales for the company.

Technology Stack & Development Environment: The project utilizes modern web development technologies. The frontend is built using HTML5, CSS3, and JavaScript, with React.js for building a responsive user interface. On the backend, Node.js and the Express framework are used for API development, while MySQL is used for database management. The payment gateway will be integrated using Stripe. The website will be hosted on AWS cloud to ensure scalability and high availability. This tech stack ensures a solid foundation for the project and addresses the scalability, security, and performance requirements.

4. Requirements Analysis

Functional Requirements:

The system should allow users to:

  • Browse and search for products.
  • View detailed product information.
  • Add products to the shopping cart and proceed to checkout.
  • Complete transactions securely via multiple payment methods.
  • Manage user accounts (registration, login, password reset).
  • Manage product listings and orders (admin functionality).
  • Provide real-time updates on product availability via an inventory management system.

Non-Functional Requirements:

  • Performance: Website load time should be under 2 seconds.
  • Security: The system must comply with data protection regulations, implement SSL encryption, and provide secure payment integration (PCI-DSS compliance).
  • Scalability: The system should support high traffic and allow easy addition of new features in the future.
  • Reliability: The website should guarantee 99.9% uptime, with minimal downtime for maintenance.

User Requirements:

The end-users expect:

  • A fast and responsive website with minimal load times.
  • Easy navigation and intuitive browsing experience.
  • Secure payment processing and data protection during transactions.
  • A smooth checkout process with multiple payment options.
  • Accessibility across various devices (desktop, tablet, and mobile).
  • Real-time updates on product availability and inventory.

Constraints:

  • Budget limitations may restrict the inclusion of advanced features in the initial phases.
  • The project must be completed within a 6-month timeframe, with a final deadline of July 19, 2025.
  • Compliance with all relevant data protection laws (GDPR, PCI-DSS, etc.) is mandatory.

5. System Design

Architecture Design:

The system will have a modular architecture comprising the following key components:

  • Frontend: A responsive web interface using modern HTML, CSS, and JavaScript frameworks.
  • Backend: A secure RESTful API and database integration, handling user accounts, product listings, and transactions.
  • Payment Gateway: Integrated third-party payment system for secure transactions.
  • Inventory Management: Real-time updates for product availability.

Database Design:

The database will include the following key tables:

  • Users: Stores user information (username, email, password hash, role).
  • Products: Stores product details (name, description, price, stock quantity, category).
  • Orders: Stores order details (order ID, user ID, product IDs, payment status).
  • Transactions: Stores transaction records (transaction ID, payment method, status, amount).

Interface Design:

The interfaces between modules will include:

  • Frontend to Backend API: HTTP/RESTful calls, returning JSON data.
  • Backend to Database: SQL queries for data retrieval and manipulation.
  • Payment Gateway Integration: Secure API calls to third-party payment processors (e.g., Stripe, PayPal).

User Interface Design:

The user interface will focus on the following principles:

  • Minimalistic design with clear navigation.
  • Fast loading times and mobile responsiveness.
  • Clear product display with filtering options.
  • Simple checkout process with an intuitive cart.

Technology Stack:

The following technologies will be used for development:

  • Frontend: HTML5, CSS3, JavaScript (React or Vue.js for dynamic components).
  • Backend: Node.js or Python (Django or Flask for API development).
  • Database: MySQL or PostgreSQL.
  • Payment Gateway: Stripe or PayPal for secure payments.
  • Hosting: Cloud-based platform (AWS or Google Cloud).
